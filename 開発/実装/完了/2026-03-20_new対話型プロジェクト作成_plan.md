## フロントエンド計画

### 変更ファイル一覧

| ファイル | 変更種別 | 内容 |
|---------|---------|------|
| `components/chat/ToolActivity.tsx` | 新規 | ツール実行表示コンポーネント |
| `components/chat/ChatMessage.tsx` | 修正 | tool 表示を ToolActivity に委譲 |
| `hooks/useProjectChat.ts` | 修正 | complete 時にプロジェクトパスを保持する state 追加 |
| `app/new/page.tsx` | 修正 | 完了画面に VS Code で開くボタン追加 |

### Step 1: ToolActivity コンポーネント作成

**ファイル:** `devtools/frontend/src/components/chat/ToolActivity.tsx`（新規）

- Props: `toolName?: string`, `content: string`（toolName は optional、ChatItem.toolName と一致）
- ツール名をバッジ風に表示、内容を小さいテキストで表示
- toolName が undefined の場合はバッジ非表示
- Client Component

### Step 2: ChatMessage の tool 表示を委譲

**ファイル:** `devtools/frontend/src/components/chat/ChatMessage.tsx`

- `case "tool"` の表示部分を `ToolActivity` コンポーネントに差し替え

### Step 3: useProjectChat にプロジェクトパス保持を追加

**ファイル:** `devtools/frontend/src/hooks/useProjectChat.ts`

- State 追加: `createdPath: string | null`
- `complete` ケースで `event.result.output` から正規表現 `/生成先:\s*(\/Users\/user\/[a-zA-Z0-9_-]+)/` でパスを抽出して保持
  - SKILL.md Step 11 の完了メッセージに `生成先: /Users/user/<プロジェクト名>/` が含まれる
  - 「生成先:」プレフィックスを含めてマッチさせ、他の行（`cd /Users/user/...` 等）との誤マッチを防ぐ
  - マッチしない場合は `createdPath = null` のまま（VS Code ボタンが disabled になる）
- `UseProjectChatReturn` に `createdPath` を追加
- `reset` で `createdPath` を null にリセット

### Step 4: 完了画面に VS Code ボタン追加

**ファイル:** `devtools/frontend/src/app/new/page.tsx`

- `openInVSCode` を `@/lib/createApi` から import して再利用
- `useProjectChat` から `createdPath` を取得
- `phase === "complete"` の画面に「VS Codeで開く」ボタンを追加
- `createdPath` が null の場合はボタンを `disabled` にする（パス抽出失敗時のフォールバック）
- ボタンの状態管理（isOpening, openError）は page 内で useState
- `openInVSCode` 呼び出しを try/catch で囲み、失敗時はボタン下にエラーメッセージを表示

---

## 設計判断

| 判断 | 選択 | 理由 |
|-----|------|------|
| ToolActivity の粒度 | 軽量な表示コンポーネント | チャットUIではツール情報は補助的 |
| VS Code ボタン | page.tsx に直接実装 | 完了画面のみで使う1箇所のUI |
| パス取得方法 | output テキストから正規表現で抽出 | バックエンドの /init 完了メッセージに生成先パスが含まれる |
| 旧フォームUI | Phase 1 では残す | openInVSCode は引き続き使用。Phase 2 で削除 |

## 実装ステップ

1. `ToolActivity.tsx` 新規作成
2. `ChatMessage.tsx` の tool 表示を ToolActivity に委譲
3. `useProjectChat.ts` に `createdPath` state 追加
4. `page.tsx` に VS Code ボタン追加
