# プロジェクトポート自動割り当て 実装計画

## Context

生成プロジェクトが全て同じポート（8080, 3000等）を使うため、複数プロジェクトを同時起動できない。
プロジェクトごとにランダムな4桁ポートを割り当てて固定する。

検討資料: `開発/検討中/2026-03-20_プロジェクトポート自動割り当て.md`

## 実装方針

- 先頭桁を固定してサービスの識別性を保つ:
  - Backend: 8xxx
  - Frontend: 3xxx
  - PostgreSQL: 5xxx
  - MinIO API: 9xxx（コンソール: +1）
  - Redis: 6xxx
- 下3桁をランダム生成（100〜999）
- 予約済みポート（8080, 8888, 3000, 3333, 5432, 9000, 6379）は除外
- 生成時に使用中でないか確認
- テンプレートにプレースホルダーを埋め込み、/init 時に置換

## テンプレート変更計画

### プレースホルダー定義

| プレースホルダー | 用途 | 例 |
|----------------|------|-----|
| `{{PORT_BACKEND}}` | バックエンドポート | 8347 |
| `{{PORT_FRONTEND}}` | フロントエンドポート | 3347 |
| `{{PORT_DB}}` | PostgreSQLポート | 5347 |
| `{{PORT_MINIO}}` | MinIO APIポート | 9347 |
| `{{PORT_MINIO_CONSOLE}}` | MinIO コンソールポート | 9348 |
| `{{PORT_REDIS}}` | Redisポート | 6347 |

### 変更ファイル一覧

| ファイル | 変更内容 |
|---------|---------|
| `templates/base/docker-compose.yml` | 8080→`{{PORT_BACKEND}}`, 3000→`{{PORT_FRONTEND}}` |
| `templates/base/backend/.env.example` | PORT=`{{PORT_BACKEND}}`, ALLOWED_ORIGIN の 3000→`{{PORT_FRONTEND}}` |
| `templates/base/frontend/next.config.ts` | localhost:8080→localhost:`{{PORT_BACKEND}}` |
| `templates/base/Makefile` | 8080→`{{PORT_BACKEND}}`, 3000→`{{PORT_FRONTEND}}` |
| `templates/with-db/docker-compose.yml` | 5432→`{{PORT_DB}}`, 8080→`{{PORT_BACKEND}}`, 3000→`{{PORT_FRONTEND}}` |
| `templates/with-storage/docker-compose.yml` | 9000→`{{PORT_MINIO}}`, 9001→`{{PORT_MINIO_CONSOLE}}` |
| `templates/with-redis/docker-compose.yml` | 6379→`{{PORT_REDIS}}` |

### 注意: コンテナ内ポートは変えない

docker-compose の `ports:` はホスト側のみ変更。コンテナ内部は元のまま:
```yaml
ports:
  - "{{PORT_BACKEND}}:8080"   # ホスト側だけ変える
```

環境変数 `PORT=8080` もコンテナ内なので変えない。変えるのは:
- ホスト側ポートマッピング
- localhost:XXXX の参照箇所（.env, next.config.ts, Makefile）

## SKILL.md 変更計画

### 変更ファイル

| ファイル | 変更内容 |
|---------|---------|
| `.claude/skills/init/SKILL.md` | Step 3（テンプレートコピー）の後にポート割り当て + 置換ステップを追加 |

### 追加するステップ: ポート割り当て

Step 4（プレースホルダー置換）の中で、`{{PROJECT_NAME}}` と一緒に `{{PORT_xxx}}` も置換する。

ポート生成ルール:
1. 下3桁をランダム生成（100〜999）
2. 各サービスの先頭桁を付与（Backend=8, Frontend=3, DB=5, MinIO=9, Redis=6）
3. 予約ポート（8080, 8888, 3000, 3333, 5432, 9000, 9001, 6379）に該当したら再生成
4. `lsof -ti:XXXX` で使用中チェック、使用中なら再生成

SKILL.md の Step 4 の sed コマンドに追加:
```bash
# ポート割り当て（例: 下3桁 = 347）
PORT_SUFFIX=$((RANDOM % 900 + 100))
PORT_BACKEND=8${PORT_SUFFIX}
PORT_FRONTEND=3${PORT_SUFFIX}
PORT_DB=5${PORT_SUFFIX}
PORT_MINIO=9${PORT_SUFFIX}
PORT_MINIO_CONSOLE=$((9${PORT_SUFFIX} + 1))
PORT_REDIS=6${PORT_SUFFIX}

# 置換
find . -type f \( -name "*.go" -o ... \) \
  -exec sed -i '' \
    -e "s/{{PORT_BACKEND}}/${PORT_BACKEND}/g" \
    -e "s/{{PORT_FRONTEND}}/${PORT_FRONTEND}/g" \
    -e "s/{{PORT_DB}}/${PORT_DB}/g" \
    -e "s/{{PORT_MINIO_CONSOLE}}/${PORT_MINIO_CONSOLE}/g" \
    -e "s/{{PORT_MINIO}}/${PORT_MINIO}/g" \
    -e "s/{{PORT_REDIS}}/${PORT_REDIS}/g" \
    {} +
```

### 完了メッセージにポート情報を追加

Step 11 の完了メッセージを更新:
```
アクセス:
  フロントエンド: http://localhost:${PORT_FRONTEND}
  バックエンド API: http://localhost:${PORT_BACKEND}/api/health
```

devtools の `CreateService` でもプロジェクト生成する場合は、template.go の `ReplacePlaceholders` にポート置換を追加する必要がある。ただし Phase 1 では /init SKILL.md 経由の生成のみ対応し、Go コード側は後回し。

## 実装順序

1. テンプレートファイルのポート番号をプレースホルダーに置換（7ファイル）
2. SKILL.md にポート割り当て + 置換ロジックを追加
3. SKILL.md の完了メッセージにポート情報を追加
4. 手動テスト: /init で生成 → ポートが正しく割り当てられるか確認

---

## 検証方法

1. `/init test-port` でプロジェクト生成
2. 生成された `.env` にランダムポートが書かれていること
3. `docker-compose.yml` のポートマッピングが正しいこと
4. `make dev` でバックエンド・フロントエンドが割り当てポートで起動すること
5. 2つ目のプロジェクトを生成して、異なるポートが割り当てられること
6. 両プロジェクトを同時起動できること
