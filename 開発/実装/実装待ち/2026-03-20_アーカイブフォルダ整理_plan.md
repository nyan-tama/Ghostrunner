# アーカイブフォルダ整理 実装計画

## 概要

トップレベルの `開発/アーカイブ/` を廃止し、各サブフォルダにアーカイブを分散する。
新規プロジェクト作成時（/init）にも `開発/` フォルダ群を生成するようにする。

## 実装ステップ

### Step 1: 新フォルダ作成

```
開発/実装/完了/アーカイブ/   ← 新設
開発/資料/アーカイブ/        ← 新設
```

`開発/検討中/アーカイブ/` は既存。

### Step 2: ファイル移動

**plan/report ファイル（28件） → `開発/実装/完了/アーカイブ/`**

`開発/アーカイブ/` 内の `*_plan.md` と `*_report.md` を移動。

**それ以外（20件） → `開発/検討中/アーカイブ/`**

`開発/アーカイブ/` 内の残りのファイルを移動。

### Step 3: 旧フォルダ削除

`開発/アーカイブ/` ディレクトリを削除。

### Step 4: /init SKILL.md に開発フォルダ生成ステップを追加

**ファイル:** `.claude/skills/init/SKILL.md`

Step 8（Git 初期化）の直前に、`開発/` フォルダ群を作成するステップを追加する。

```bash
mkdir -p /Users/user/<プロジェクト名>/開発/検討中/アーカイブ
mkdir -p /Users/user/<プロジェクト名>/開発/実装/実装待ち
mkdir -p /Users/user/<プロジェクト名>/開発/実装/完了/アーカイブ
mkdir -p /Users/user/<プロジェクト名>/開発/資料/アーカイブ
```

各ディレクトリに `.gitkeep` を配置して空フォルダを Git 管理可能にする。

## 変更対象ファイル

| ファイル | 変更内容 |
|---------|---------|
| `開発/アーカイブ/*_plan.md`, `*_report.md` | `開発/実装/完了/アーカイブ/` に移動 |
| `開発/アーカイブ/*.md`（残り） | `開発/検討中/アーカイブ/` に移動 |
| `開発/アーカイブ/` | 削除 |
| `.claude/skills/init/SKILL.md` | 開発フォルダ生成ステップ追加 |
