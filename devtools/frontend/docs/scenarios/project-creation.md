# プロジェクト作成シナリオ

## 概要

Ghost Runner で新しいプロジェクトを作成するユーザー操作シナリオ。`/new` ページでプロジェクト名、概要、Data Services を指定してプロジェクトを生成する。

---

## シナリオ1: 基本的なプロジェクト作成

### 目的

Go + Next.js のプロジェクトを新規作成し、開発を開始する。

### 前提条件

- Ghost Runner が起動している
- バックエンドの API が利用可能

### 操作フロー

1. **メインページ (`/`)** でヘッダーの "New Project" リンクをクリックする
2. **New Project ページ (`/new`)** に遷移し、入力フォームが表示される
3. **Project Name** にプロジェクト名（例: `my-app`）を入力する
4. 300ms 後にバリデーションが実行され、作成先パスが緑文字で表示される
5. **Create Project** ボタンをクリックする
6. 進捗チェックリストが表示され、各ステップが順番に完了していく
7. 全ステップが完了すると、成功画面が表示される

### 成功時

- "Project Created" メッセージが表示される
- 作成されたプロジェクトの絶対パスが表示される
- "Open in VS Code" ボタンと "Create Another" ボタンが表示される

### エラー時

- エラーメッセージが赤背景で表示される
- フォームが入力値を保持した状態で再表示される
- 修正して再度 "Create Project" をクリックできる

---

## シナリオ2: Data Services 付きプロジェクト作成

### 目的

データベース、ストレージ、キャッシュを含むプロジェクトを作成する。

### 前提条件

- Ghost Runner が起動している

### 操作フロー

1. **New Project ページ (`/new`)** でプロジェクト名を入力する
2. **Description** に概要を入力する（任意）
3. **Data Services** セクションで必要なサービスにチェックを入れる
   - PostgreSQL + GORM: データベースが必要な場合
   - Cloudflare R2 / MinIO: ファイルストレージが必要な場合
   - Redis: キャッシュが必要な場合
4. **Summary** セクションで入力内容を確認する
5. **Create Project** ボタンをクリックする

### 成功時

- 選択したサービスのテンプレートが適用されたプロジェクトが作成される

---

## シナリオ3: 完了後のアクション

### 目的

作成完了後にプロジェクトを開く、または続けて別のプロジェクトを作成する。

### 前提条件

- プロジェクト作成が完了している（Complete フェーズ）

### 操作フロー（VS Code で開く）

1. **Complete 画面** で "Open in VS Code" ボタンをクリックする
2. バックエンド API 経由で VS Code が起動する
3. 作成されたプロジェクトが VS Code で開かれる

### 操作フロー（もう1つ作る）

1. **Complete 画面** で "Create Another" ボタンをクリックする
2. フォームが初期状態にリセットされて表示される
3. 新しいプロジェクトの情報を入力する

### エラー時（VS Code 起動失敗）

- エラーメッセージが赤文字で表示される

---

## シナリオ4: バリデーションエラー

### 目的

無効なプロジェクト名を入力した場合の動作を確認する。

### 操作フロー

1. **New Project ページ (`/new`)** でプロジェクト名を入力する
2. 300ms デバウンス後にバリデーション API が呼び出される
3. 名前が無効（既存のディレクトリと重複など）の場合、赤文字でエラーメッセージが表示される
4. **Create Project** ボタンは無効化される（クリックできない）
5. プロジェクト名を修正すると、再度バリデーションが実行される

### バリデーション結果

| 入力状態 | 結果 | ボタン状態 |
|---------|------|----------|
| 未入力 | 表示なし | 無効 |
| バリデーション中 | "Validating..." | 無効 |
| 有効な名前 | 作成先パス表示（緑） | 有効 |
| 無効な名前 | エラーメッセージ（赤） | 無効 |

---

## シナリオ5: エラーからの復帰

### 目的

プロジェクト作成中にエラーが発生した場合、入力を保持したまま再試行する。

### 前提条件

- プロジェクト作成中にエラーが発生した

### 操作フロー

1. 作成中にエラーが発生し、チェックリストのエラー箇所に赤いバツアイコンが表示される
2. フォームが再表示される（入力値は保持）
3. エラーメッセージが赤背景で表示される
4. 必要に応じて入力内容を修正する
5. **Create Project** ボタンをクリックして再試行する

### 成功時

- ステップが最初からやり直され、正常に完了する
