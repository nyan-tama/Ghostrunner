# 巡回ダッシュボード操作シナリオ

## プロジェクト登録

### 目的

複数プロジェクトを巡回対象として登録し、自動巡回の監視対象にする。

### 前提条件

- バックエンドに1つ以上のプロジェクトが登録されている

### 操作フロー

1. **巡回ダッシュボード** (`/patrol`) を開く
2. 「追加」ボタンをクリックしてプロジェクト選択パネルを展開する
3. 登録可能なプロジェクト一覧から巡回対象にしたいプロジェクトを選択する
4. プロジェクトカードがダッシュボードに表示される

### 成功時

- プロジェクトカードがグリッドに追加される
- ステータスバッジは「待機中」（idle）で表示される

### エラー時

- 画面上部にエラーメッセージが表示される

---

## プロジェクト解除

### 目的

不要になったプロジェクトを巡回対象から外す。

### 前提条件

- 1つ以上のプロジェクトが巡回対象に登録されている

### 操作フロー

1. 解除したいプロジェクトカードの「解除」ボタンをクリックする
2. プロジェクトカードが一覧から削除される

### 成功時

- プロジェクトカードが一覧から消える
- 全プロジェクトを解除した場合、空状態メッセージが表示される

### エラー時

- 画面上部にエラーメッセージが表示される

---

## 巡回開始・停止

### 目的

登録済みプロジェクトの自動巡回を開始または停止する。

### 前提条件

- 1つ以上のプロジェクトが巡回対象に登録されている
- SSE 接続が確立されている（接続状態インジケーターが緑）

### 操作フロー（開始）

1. 「巡回開始」ボタンをクリックする
2. ボタンが「処理中...」に変わり、一時的に無効化される
3. 巡回が開始され、各プロジェクトの状態が SSE でリアルタイム更新される
4. ボタンが「巡回停止」に切り替わる

### 操作フロー（停止）

1. 「巡回停止」ボタンをクリックする
2. ボタンが「処理中...」に変わり、一時的に無効化される
3. 巡回が停止される
4. ボタンが「巡回開始」に切り替わる

### 成功時

- プロジェクトカードのステータスバッジがリアルタイムで変化する（idle -> running -> completed 等）

### エラー時

- 画面上部にエラーメッセージが表示される

---

## 承認待ちへの回答

### 目的

巡回中に承認待ち（waiting_approval）状態になったプロジェクトに対して回答を送信し、処理を続行させる。

### 前提条件

- 巡回が実行中である
- プロジェクトが承認待ち状態になっている（ステータスバッジが「承認待ち」）

### 操作フロー

1. 承認待ちのプロジェクトカードに回答フォームが表示される
2. 以下のいずれかの方法で回答する
   - **選択肢ボタン**をクリック（単一選択モードの場合、即座に送信される）
   - **複数選択モード**の場合、選択肢を選んで「選択を送信」ボタンをクリック
   - **テキスト入力欄**に自由テキストを入力して送信（Enter キーまたは送信ボタン）
3. プロジェクトの状態が「実行中」（running）に戻る

### 成功時

- 回答フォームが消え、ステータスバッジが「実行中」に変わる
- 巡回処理が続行される

### エラー時

- 画面上部にエラーメッセージが表示される

---

## ポーリングの切り替え

### 目的

バックエンドによる定期的なプロジェクトスキャン（ポーリング）を有効化/無効化する。

### 操作フロー

1. PatrolHeader 内の「ポーリング」チェックボックスをクリックする
2. チェック ON でポーリング開始、チェック OFF でポーリング停止

### 成功時

- チェックボックスの状態が切り替わる
- ポーリング有効時、バックエンドが定期的にプロジェクトをスキャンし、SSE で状態更新を通知する

### エラー時

- 画面上部にエラーメッセージが表示される

---

## リアルタイム監視

### 目的

SSE 接続を通じて、巡回中のプロジェクト状態をリアルタイムで把握する。

### 前提条件

- ページが開かれている（SSE 接続は自動確立）

### 動作

1. ページマウント時に `EventSource` で `/api/patrol/stream` に常時接続する
2. 接続状態インジケーターで接続状況を確認できる
   - 緑：接続済み
   - 黄（点滅）：接続中
   - 赤：切断
3. 各種 SSE イベントを受信すると、該当プロジェクトカードのステータスがリアルタイムで更新される
4. 接続が切れた場合、指数バックオフで自動再接続を試行する（最大10回）

### 表示される情報

- プロジェクトのステータス（待機中 / 実行中 / 承認待ち / キュー待ち / 完了 / エラー）
- 最近のコミット（最大5件）
- 実装待ちタスク数
- エラーメッセージ
