# プロジェクト削除シナリオ

## 概要

Ghost Runner のメインページ (`/`) でプロジェクトの登録を解除するユーザー操作シナリオ。

---

## シナリオ1: プロジェクトを削除する

### 目的

不要になったプロジェクトの登録を解除し、プロジェクト一覧から除外する。

### 前提条件

- バックエンドサーバーが起動している
- 登録済みのプロジェクトが1つ以上存在する

### 操作フロー

1. **メインページ** (`/`) の左下にある「削除モード」ボタンをクリック
2. コマンド入力エリアが非表示になり、**プロジェクト削除リスト**（ProjectDeleteList）が表示される
3. 削除したいプロジェクトの横にある「削除」ボタンをクリック
4. 確認ダイアログが表示される（「プロジェクト「XXX」を削除しますか？」）
5. 「OK」をクリック
6. 削除処理が実行され、ボタンが「削除中...」に変化する
7. 削除完了後、プロジェクト一覧が再取得され、削除したプロジェクトが一覧から消える

### 成功時

- プロジェクトが一覧から削除される
- プロジェクト一覧が自動的に再取得される

### エラー時

- `alert` でエラーメッセージが表示される（「プロジェクトの削除に失敗しました: XXX」）
- プロジェクト一覧は変更されない

---

## シナリオ2: 選択中のプロジェクトを削除する

### 目的

コマンド実行用に選択しているプロジェクトを削除し、選択状態をリセットする。

### 前提条件

- バックエンドサーバーが起動している
- プロジェクトが選択されている状態

### 操作フロー

1. **メインページ** (`/`) でプロジェクトを選択した状態で「削除モード」ボタンをクリック
2. プロジェクト削除リストが表示される
3. 選択中のプロジェクトの「削除」ボタンをクリック
4. 確認ダイアログで「OK」をクリック
5. 削除完了後、プロジェクト選択がリセットされる（projectPath が空になる）
6. 「削除モード」ボタンをクリックして通常モードに戻る
7. Project Path ドロップダウンが未選択状態に戻っている

### 成功時

- プロジェクトが削除され、プロジェクト選択がリセットされる
- 通常モードに戻ると、ドロップダウンから別のプロジェクトを選択できる

---

## シナリオ3: 削除をキャンセルする

### 目的

誤って削除ボタンをクリックした場合に、削除操作をキャンセルする。

### 前提条件

- 削除モードが有効

### 操作フロー

1. プロジェクト削除リストで「削除」ボタンをクリック
2. 確認ダイアログが表示される
3. 「キャンセル」をクリック
4. 何も変更されず、プロジェクト削除リストがそのまま表示される

### 成功時

- プロジェクトは削除されない
- 一覧に変更なし

---

## シナリオ4: 削除モードから通常モードに戻る

### 目的

プロジェクト削除が不要になった場合、通常のコマンド入力画面に戻る。

### 前提条件

- 削除モードが有効

### 操作フロー

1. 画面左下の「削除モード ON」ボタンをクリック
2. プロジェクト削除リストが非表示になる
3. コマンド入力エリア（CommandForm）が再表示される

### 成功時

- 通常のコマンド入力画面に戻る
- プロジェクト選択やコマンド設定は削除モード切替前の状態を保持
