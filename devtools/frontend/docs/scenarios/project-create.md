# プロジェクト作成シナリオ

## 概要

Ghost Runner の `/new` ページで新規プロジェクトを作成するユーザー操作シナリオ。

---

## シナリオ1: 基本的なプロジェクト作成

### 目的

Go + Next.js の基本構成でプロジェクトを新規作成する。

### 前提条件

- バックエンドサーバーが起動している

### 操作フロー

1. **メインページ** (`/`) のヘッダーにある「New Project」リンクをクリック
2. **New Project ページ** (`/new`) に遷移し、フォームが表示される
3. **Project Name** にプロジェクト名を入力する（例: `my-awesome-app`）
4. 300ms 後にバリデーションが実行され、作成先パスが緑色で表示される
5. （任意）**Description** にプロジェクトの概要を入力する
6. **Summary** セクションで入力内容を確認する
7. **Create Project** ボタンをクリックする
8. 進捗表示に切り替わり、10ステップが順次実行される
9. 全ステップが完了すると、完了画面が表示される

### 成功時

- 完了画面にプロジェクトパスが表示される
- 「Open in VS Code」ボタンで VS Code を起動できる
- 「Create Another」ボタンで別のプロジェクトを作成できる

### エラー時

- エラーメッセージが赤枠で表示される
- フォームが再表示され、入力値は保持される
- 修正後に「Create Project」ボタンで再実行できる

---

## シナリオ2: データサービス付きプロジェクト作成

### 目的

PostgreSQL、オブジェクトストレージ、Redis など、データサービスを含むプロジェクトを作成する。

### 前提条件

- バックエンドサーバーが起動している

### 操作フロー

1. **New Project ページ** (`/new`) にアクセスする
2. **Project Name** にプロジェクト名を入力する
3. **Data Services** セクションで利用するサービスのチェックボックスを選択する
   - PostgreSQL + GORM: データベースとマイグレーション
   - Cloudflare R2 / MinIO: オブジェクトストレージ
   - Redis: インメモリキャッシュとセッションストア
4. **Summary** セクションで選択したサービスが表示されることを確認する
5. **Create Project** ボタンをクリックする
6. テンプレートのコピーから依存関係のインストールまで、選択したサービスに応じた構成でプロジェクトが作成される

### 成功時

- 選択したサービスの設定ファイルやテンプレートが含まれたプロジェクトが作成される

---

## シナリオ3: バリデーションエラーへの対応

### 目的

プロジェクト名が無効な場合に、エラーを確認して修正する。

### 前提条件

- バックエンドサーバーが起動している

### 操作フロー

1. **Project Name** に無効なプロジェクト名を入力する（例: 既存のプロジェクト名、不正な文字を含む名前）
2. 300ms 後にバリデーションが実行される
3. エラーメッセージが赤色で表示される
4. **Create Project** ボタンが無効化される
5. プロジェクト名を有効な値に修正する
6. 作成先パスが緑色で表示され、ボタンが有効化される

### 成功時

- 修正後のプロジェクト名でプロジェクトを作成できる

---

## シナリオ4: エラー後のリトライ

### 目的

プロジェクト作成中にエラーが発生した場合、入力値を保持したまま再実行する。

### 前提条件

- バックエンドサーバーが起動している

### 操作フロー

1. フォームに入力して **Create Project** ボタンをクリックする
2. 進捗表示の途中でエラーが発生する（例: 依存関係のインストール失敗）
3. エラーが発生したステップが赤色のバツマークで表示される
4. エラーメッセージが赤枠で表示され、フォームが再表示される
5. 入力値（プロジェクト名、説明、サービス選択）が保持されていることを確認する
6. 必要に応じて入力値を修正する
7. **Create Project** ボタンをクリックして再実行する

### 成功時

- リトライが成功し、完了画面が表示される

### エラー時

- 再度エラーが表示され、同様にリトライできる

---

## シナリオ5: 完了後に別のプロジェクトを作成

### 目的

プロジェクト作成後に、続けて別のプロジェクトを作成する。

### 前提条件

- プロジェクト作成が完了している（完了画面が表示されている）

### 操作フロー

1. 完了画面の **Create Another** ボタンをクリックする
2. フォームがリセットされ、空のフォームが表示される
3. 新しいプロジェクトの情報を入力する
4. **Create Project** ボタンをクリックして作成する

### 成功時

- 新しいプロジェクトが作成され、完了画面が表示される

---

## シナリオ6: VS Code でプロジェクトを開く

### 目的

作成したプロジェクトを VS Code で開いて開発を開始する。

### 前提条件

- プロジェクト作成が完了している（完了画面が表示されている）
- VS Code がインストールされ、`code` コマンドが使用可能

### 操作フロー

1. 完了画面の **Open in VS Code** ボタンをクリックする
2. ボタンが "Opening..." に変わり、バックエンド API が呼び出される
3. VS Code がプロジェクトのパスで起動する

### 成功時

- VS Code がプロジェクトディレクトリで開く

### エラー時

- エラーメッセージが赤色でボタン下に表示される（例: VS Code が見つからない）
