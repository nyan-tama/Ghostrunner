# Ghostrunner

Claude Code 用フルスタック開発フレームワーク。
AIと対話するだけで、Go + Next.js のWebアプリが自動生成される。

## 何ができるか

1. **ブラウザで `http://localhost:3333/new` にアクセス**
2. **「予約管理システムを作りたい」と入力**
3. **AIがMVPを提案 → 承認 → 自動でプロジェクト生成・実装**
4. **動くWebアプリが完成**

その後は VS Code で `/fullstack` を使って機能追加していく。

## 構成

```
Ghostrunner/
|-- .claude/
|   |-- agents/       # 23エージェント（Go/Next.js/PostgreSQL/ストレージ/Redis/運用）
|   |-- skills/       # 13スキル（/init, /fullstack, /plan, /discuss 等）
|   |-- settings.json # フック設定（コード品質チェック、フォーマッター）
|-- templates/
|   |-- base/         # Go + Next.js 基本構成
|   |-- with-db/      # PostgreSQL + GORM
|   |-- with-storage/ # Cloudflare R2 / MinIO
|   |-- with-redis/   # Redis
|-- devtools/
|   |-- backend/      # devtools API（Go + Gin, port 8888）
|   |-- frontend/     # devtools UI（Next.js, port 3333）
```

## 技術スタック

- **生成プロジェクト**: Go + Gin / Next.js + React + TypeScript + Tailwind CSS
- **devtools**: Go + Gin / Next.js
- **オプション**: PostgreSQL + GORM / Cloudflare R2 / Redis
- **外部アクセス**: Tailscale

## セットアップ

### 1. 前提条件

- macOS
- [Homebrew](https://brew.sh/)
- Git / Go / Node.js
- [Claude Code](https://docs.anthropic.com/en/docs/claude-code) がインストール済み

### 2. Ghostrunner のインストール

```bash
git clone https://github.com/nyan-tama/Ghostrunner.git
cd Ghostrunner
cd devtools/frontend && npm install && cd ../..
```

### 3. 起動

```bash
make dev
```

- devtools UI: http://localhost:3333
- devtools API: http://localhost:8888

### 4. プロジェクト作成

ブラウザで http://localhost:3333/new にアクセスし、AIと対話してプロジェクトを作成する。

または CLI で:

```bash
claude "/init my-project"
```

## スマホからアクセスする（Tailscale）

Tailscale を使うと、iPhone/iPad から devtools にアクセスできる。

### 1. Tailscale のインストール

**Mac:**

```bash
brew install --cask tailscale-app
```

メニューバーの Tailscale アイコンからログイン。

```bash
# 接続確認
tailscale status

# 自分の IP を確認
tailscale ip -4
# => 100.x.x.x
```

**iPhone / iPad:**

App Store で「Tailscale」をインストールし、Mac と同じアカウントでログイン。

### 2. アクセス

Mac で `make dev` を実行した状態で、iPhone の Safari から:

```
http://<Mac の Tailscale IP>:3333
```

これで devtools UI が表示される。

### 3. セキュリティ

- Tailscale は VPN なので、Tailnet に参加していないデバイスからはアクセスできない
- カフェ等の公共 Wi-Fi が気になる場合は、macOS のファイアウォールをオンにする
  - 「システム設定 → ネットワーク → ファイアウォール」
  - Tailscale 経由のアクセスはファイアウォールを通過するので影響なし

### 4. Tailscale Funnel（インターネット公開、オプション）

Tailnet 外のインターネットからアクセスしたい場合:

```bash
# フロントエンドをインターネットに公開
tailscale funnel --bg 3333

# 停止
tailscale funnel 3333 off
```

公開 URL: `https://<hostname>.tail<tailnet>.ts.net/`

## 主要コマンド

### Makefile（Ghostrunner ルート）

```bash
make dev      # devtools を起動（停止→起動）
make stop     # devtools を停止
make health   # ヘルスチェック
```

### Claude Code スキル（VS Code ターミナル）

```bash
/init         # 新規プロジェクト作成（対話型）
/discuss      # アイデアの相談・検討
/plan         # 実装計画の作成
/fullstack    # 計画に基づく自動実装（レビュー・テスト込み）
/stage        # ステージング環境にデプロイ
/release      # 本番リリース
```

## ポート一覧

| サービス | ポート | 用途 |
|---------|--------|------|
| devtools フロントエンド | 3333 | 進捗ビューア・プロジェクト作成UI |
| devtools バックエンド | 8888 | devtools API |
| 生成プロジェクト フロントエンド | 3xxx | 各プロジェクトごとにランダム |
| 生成プロジェクト バックエンド | 8xxx | 各プロジェクトごとにランダム |
| PostgreSQL | 5xxx | 各プロジェクトごとにランダム |
| MinIO | 9xxx | 各プロジェクトごとにランダム |
| Redis | 6xxx | 各プロジェクトごとにランダム |

生成プロジェクトのポートは先頭桁固定 + 下3桁ランダムで自動割り当て。
複数プロジェクトを同時起動しても衝突しない。
